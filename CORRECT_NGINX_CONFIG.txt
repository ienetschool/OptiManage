# CORRECT NGINX CONFIGURATION FIX

## The Problem: Duplicate location "/" blocks
Your Plesk already has a location / block, so adding another one causes the duplicate error.

## SOLUTION: Use a different approach

Instead of location /, use this configuration in Plesk:

```nginx
location /api/ {
    proxy_pass http://127.0.0.1:8080/api/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}

location ~ ^/(patients|appointments|prescriptions|doctors|dashboard|auth).*$ {
    proxy_pass http://127.0.0.1:8080$request_uri;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

## ALTERNATIVE SOLUTION: Clear existing and use simple config

1. Remove all existing nginx directives in Plesk
2. Add only this:

```nginx
try_files $uri @proxy;

location @proxy {
    proxy_pass http://127.0.0.1:8080;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

## SIMPLEST FIX: Use subdirectory approach

Add this to nginx directives (this won't conflict):

```nginx
location /app/ {
    proxy_pass http://127.0.0.1:8080/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

Then access your app at: https://opt.vivaindia.com/app/
